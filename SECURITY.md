# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅        |
| < 1.0   | ❌        |

## Reporting a Vulnerability

If you discover a security vulnerability in this package — including anything related to permission resolution, the super admin bypass, the middleware, or the bundled admin UI — please **do not** open a public GitHub issue.

Instead, report it privately by emailing:

**[security@tchenry.com](mailto:security@tchenry.com)**

Please include:

- A description of the vulnerability and its potential impact.
- Steps to reproduce, or a minimal proof of concept.
- The package version, PHP version, and Laravel version affected.

You should expect an initial response within **72 hours**. We'll keep you updated as the issue is investigated and resolved, and we'll credit you in the release notes unless you prefer to remain anonymous.

## Disclosure Process

1. The report is triaged and confirmed.
2. A fix is prepared and tested privately.
3. A new patch or minor version is released with the fix.
4. The vulnerability is disclosed publicly in the [changelog](CHANGELOG.md) and, where relevant, via a GitHub Security Advisory — after the fix is available.

Thank you for helping keep this package and its users safe.
